LINE RX
Line Balance Methodology — Field Reference
Version 1.0 · For use with ProcessDx and Line Rx · April 2026
What this is
Line Rx is a demand-driven line balance methodology. It starts with actual time study data, accounts for mixed-model demand, identifies the constraint, removes recoverable waste, and rebalances remaining work content — producing a defensible future state and a quantified savings case.
When to Run Line Rx
Trigger
Line is chronically over takt, understaffed, overstaffed, or demand mix has shifted and line balance has not been revisited.
Not a replacement for
Process stability. Solve ergo, variation, and equipment reliability before rebalancing. Balancing an unstable process locks instability in.
Output
Future state work assignment per station, headcount case, and annualized savings. One-page summary suitable for leadership review.
Prerequisites
Time studies
ProcessDx study complete for each station and each process option. Abnormal cycles excluded. All steps classified VA or Waste.
Demand data
Required units per shift per option — known and confirmed with planning. This is not estimated. Wrong units produce wrong takt.
Available time
Confirmed shift length and PF&D allowance. Defaults: 455 min / 15% PF&D = 386.75 min effective. Adjust to actual.
Labor rate
Straight-time labor rate for savings calculation. Pulled from ProcessDx setup if entered. Falls back to Line Rx global input.
The 7-Step Method
1
Study — Complete ProcessDx for Every Station
One ProcessDx study per station per process option. If a station runs Style A and Style B, that is two JSON files. If one operator does manual and auto processes, those are two separate studies.
Exclude abnormal cycles before exporting. Talking to a supervisor is not part of the standard method.
Classify every step: VA or Waste. No incidental category. Calling a step incidental gives permission to keep waste permanently.
Flag ergonomic risks. Fix the body before you time the method.
Export JSON after completing Gate 6. That file is the input to Line Rx.
Output: One ProcessDx JSON per station per option
2
Build — Assemble Stations in Line Rx
Open Line Rx Station Builder. Create one station for each position in the line. Attach all ProcessDx JSON files for that station. Enter required units per shift for each option.
Station name = the position, not the product. "Screen Build" not "L2 Roll Auto."
Units come from the planning system. Confirm with the scheduler before entering.
Mark machine-constrained stations. Headcount math cannot be applied there without capital approval.
Set Available Time and PF&D in Line Setup. Match what you told the study operators.
Output: Weighted CT and station takt calculated for all stations
3
Read — Understand Current State
Switch to Balance View. Read the chart before drawing any conclusions. The constraint is the station with the largest gap between its weighted CT and its takt — not the tallest bar.
Check option overload flags. A flag means one process option CT exceeds takt — that is a scheduling problem, not necessarily a staffing problem.
Read utilization by station. Under 70% with no capacity problem = opportunity to reduce headcount. Over 100% = must act before this shift runs long.
Headcount = Total Time Consumed ÷ Effective Available. This is a floor — never a ceiling. Machine constraints and physical layout may prevent actualization.
Write down the constraint name and gap before switching to Future State. That is your problem statement.
Output: Constraint identified, current state documented, problem statement confirmed
4
Remove — Eliminate Recoverable Waste
Switch to Future State. Open Yamazumi View. Work through each station at the constraint first. Remove waste elements that can be eliminated. Every removal flows to the Savings Column automatically.
Follow the CLEAR sequence: Cut → Link → Exchange → Add → Reduce. Never skip to Reduce.
Remove only what you can actually eliminate in the current state or the agreed kaizen scope. Do not remove waste you are hoping to fix later.
Stop when the constraint station is at or under takt. Additional waste removal is still valuable but the balance problem is solved.
If waste removal alone cannot close the gap — the remaining CT is all VA. Move to Step 5.
Output: Future state CT updated, Savings Column shows time and $ recovered
5
Rebalance — Redistribute Work Content
If the constraint station is still over takt after waste removal, shift VA work content to adjacent stations with available capacity. Use the → move function in the Yamazumi view.
Only move VA elements. Moving waste to another station does not remove it — it just relocates the problem.
Check the destination station's utilization before moving. You cannot add to a station already at 90%+ without creating a new constraint.
Physical adjacency matters. The tool does not know if operator B can physically reach the work you are moving from operator A.
A move that closes the constraint but opens a new one is not a solution. Check the line balance chart after each move.
Output: All stations at or under takt in future state
6
Validate — Read the Future State
Switch to Output View. Review the comparison chart, the work assignment cards, and the savings summary. Challenge the numbers before presenting them.
If headcount drops, confirm which stations are machine-constrained. A constrained station cannot actualize a headcount reduction regardless of utilization math.
Check that no moved element creates a physical or sequencing conflict in the real cell.
Savings are calculated at current demand and current labor rate. If demand is seasonal or rate changes are expected, note it before presenting.
The future state is a hypothesis, not a plan. It becomes a plan when standardized work is written and operators are trained to it.
Output: Validated future state, headcount case, savings case
7
Export — Convert to Action
Print the Output view. Work assignment cards become the starting point for new standardized work. The comparison chart and savings summary go to the wall at kaizen close-out and to leadership for the ROI case.
Each work assignment card shows: station, future CT vs takt, headcount required, and every work element assigned to that station.
These cards are not standardized work documents. They define what tasks belong at each station. Full standardized work requires method, sequence, and training.
The savings number is your ROI case. Break-even is modest. The system pays for itself if it closes even a fraction of the identified gap.
Kaizen is not complete until the new method is running and stable. The card on the wall is the target, not the result.
Output: Printable work cards, comparison chart, savings case for leadership
Key Formulas
Effective Available
AvailMin × (1 − PF&D%) × 60
455 min × 85% × 60 = 23,205s
This is the denominator for takt and headcount.
Station Takt
Effective Available ÷ Total Units
23,205s ÷ 360 units = 64.5s per unit
Recalculates automatically when units change.
Weighted Average CT
Σ(Units × CT) ÷ Σ Units
(270 × 75s) + (90 × 104s) ÷ 360 = 82.5s
This is the bar height on the Yamazumi.
Headcount Required
Total Time Consumed ÷ Eff. Available
Σ(Units × CT) ÷ 23,205s = 1.4
This is a floor. Machine constraints and layout may limit actualization.
Time Recovered / Shift
Σ(Removed Step CT × Units)
Per removed element. Sum across all stations. This is what disappears from the schedule when the method changes.
Annual Savings
(Time Recovered ÷ 3600) × Rate × Shifts
At current demand and current labor rate. Document assumptions before presenting to leadership.
How to Read the Signals
SignalWhat It MeansWhat to Do
OVER TAKT Station weighted CT exceeds its station takt. At current demand and CT, this station cannot produce fast enough in the available time. Step 4 first: remove waste. If still over after waste removal, Step 5: move VA content to underloaded stations.
OPTION OVERLOAD One process option's CT exceeds takt — but the weighted average may still be OK. This is a scheduling sequencing problem, not a headcount problem. Flag for scheduling. Avoid concentrating high-CT options consecutively. This is the casement WW scenario.
AT LIMIT Gap is 0–5 seconds. Station is technically under takt but has no buffer for variation, changeover, or short stops. Leave in current state. Monitor for variation. Do not add work content to this station.
constrained Machine or physical layout limits operator count to 1 regardless of utilization math. Headcount goal is 1. Utilization over 100% at this station requires a capital, layout, or takt discussion — not a staffing fix.
Moved In ← A work element was moved to this station from another station during rebalancing. Confirm physical adjacency is possible. Confirm the operator can reach and perform this task. This is a hypothesis until proven on the floor.
OK Station weighted CT is under takt. Station can meet demand at current method and headcount. If utilization is significantly under (below 70%), consider whether this station can absorb work from overloaded neighbors.
Non-Negotiable Principles
Diagnose before you treat.
Do not begin removing waste or rebalancing until the current state is documented and the constraint is confirmed. The constraint drives the sequence of decisions.
% time is an output, not an input.
Do not estimate time splits. Enter the actual units required per option. Line Rx calculates the split from demand and CT. An estimated split produces a wrong takt and a wrong line balance.
Never rebalance an unstable process.
High variation steps (CV > 30%) in the ProcessDx study indicate no standard method exists. Rebalancing around an unstable step locks instability into the new design. Stabilize first.
Moving waste between stations is not rebalancing.
Only move VA elements in Step 5. Moving waste to a downstream station relocates the problem. Remove waste in Step 4, then rebalance what remains.
Headcount math requires a constraint check.
If the math says 1.4 operators, the answer is not "add 0.4 of a person." The answer is: is there room to eliminate 40% of one operator's time through waste removal, or does a second operator need to be partially deployed? Machine-constrained stations cannot actualize headcount reductions without capital.
The card on the wall is a target, not a result.
Future state is a hypothesis. It becomes a result when the new method is running, stable, and confirmed by repeated observation. Close-out starts the work, it does not complete it.
